# DOC-001: Create New Document

## Workflow ID

DOC-001

## Workflow Name

Create New Document

## Purpose

To create a new technical document using approved templates, metadata standards, and repository structure.

## Trigger

A new documentation need is identified.

Examples:

* New feature
* New API
* New system
* New procedure
* New architecture decision
* New incident

## Inputs

* Document Title
* Document Type
* Owner
* Team
* Purpose
* Audience

## Roles

### Author

Creates and maintains the document.

### Reviewer

Reviews technical accuracy and completeness.

### Approver

Approves publication of the document.

## Process

### Step 1: Identify Documentation Need

Determine whether new documentation is required.

### Step 2: Select Document Type

Choose one of:

* User Guide
* Runbook
* SOP
* ADR
* API Documentation
* Troubleshooting Guide

### Step 3: Create Document

Create a new markdown file in the appropriate repository location.

### Step 4: Complete Metadata

Populate required metadata fields.

### Step 5: Write Content

Complete all required sections.

### Step 6: Self-Review

Verify content accuracy and completeness.

### Step 7: Submit Pull Request

Create a pull request for review.

### Step 8: Review and Approval

Obtain required approvals.

### Step 9: Publish

Merge changes and publish documentation.

## Outputs

* New documentation page
* Review history
* Published content

## Success Criteria

* Template used
* Metadata complete
* Content reviewed
* Approval obtained
* Published successfully
